## Firebase free-tier risk review
If Blaze is turned off and the project is forced back to Spark, the functions-backed features in this repo are expected to break first:
- Certificate entry route rewrite `/lms/certificate`
- Certificate OG image route `/lms/certificate-image`
- Firestore trigger-based leaderboard sync from points ledger
- User-created automation email flow
- Resend webhook processing

Features that are more likely to keep working on Spark if usage stays inside free quota:
- Firebase Hosting for static pages
- LMS frontend UI
- Firebase Auth
- Firestore reads/writes within Spark limits
- Client-side certificate viewer page that does not depend on Cloud Functions

## Recommended next passes
1. Move the presentation deck onto the same theme/lang shell or intentionally isolate it as Bangla-only with a clear rationale.
2. Continue source-level Bangla cleanup file-by-file instead of relying on normalization debt.
3. Decide whether multilingual SEO should stay toggle-based or move to real language-specific URLs.
4. If Spark-only operation is required, replace functions-backed certificate share and trigger flows with static/client-safe alternatives.
